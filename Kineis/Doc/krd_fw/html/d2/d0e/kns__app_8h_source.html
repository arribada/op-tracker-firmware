8 /**
9  * @page kns_app_page APP task
10  *
11  * This page is presenting two APPlication task example which exercices the Kineis stack SW
12  * (\ref KNS_APP) in a way to send user data to the Kineis network..
13  *
14  * Two applications tasks are available:
15  * * A standalone app (\ref KNS_APP_stdln_loop) is sending one user message (once or 4 retransmit each 60s)
16  * * A GUI app (\ref KNS_APP_gui_loop) is dedicated to communicate with the Kinéis Device Interface
17  * (KDI) through AT commands (\ref mgr_at_cmd_page) over an UART link (9600 bauds, 8, N, 1).
18  *
19  * Those application needs some configurations from user such as:
20  * * Credentials (ID, ADDR, Secret key)
21  * * Radio configuration (UL frequency and modulation)
22  * * Kineis protocol configuration (a.k.a. MAC profile)
23  * Refer to "getting started" section for details.
24  *
25  * In case of GUI APP, the format of an AT command is like:
26  * * "<pattern>=?\r\n" (e.g. "AT+PING=?\r\n") to get information
27  * * "<pattern>=<params>\r\n" (e.g. "AT+TX=<user_data>\r\n") to set information
28  * The list of supported AT commands is available through \ref cas_atcmd_list_array.
29  *
30  * Communication between tasks (APP and MAC) in done through events posted in OS Queues.
31  * By the end, those applications shows how to:
32  * * Invoke the kineis stack to send USER DATA
33  * * Wait notification from the Kineis stack once the transmission is completed
34  *
35  * @section kns_app_subpages Sub-pages
36  *
37  * * @subpage mgr_at_cmd_page
38  */

52 /** @brief This fct is a very basic standalone application
53  *
54  * This application is posting some send-data event to MAC layer of Kineis software stack in loop as
55  * long as there is no error.
56  *
57  * Depending on the defined MAC protocol:
58  * * BASIC: APP is trying to send 2 data at start but kineis stack can only accept the first one.
59  * This data is sent once only, then is goes into lowpower mode.
60  * * BLIND: APP is trying to send 5 data at start but 4 are accepted. Each data is re-transmitted
61  * (retx) several times periodically as per the configuration of the protocol. Once a data is fully
62  * transmitted, APP will try to send a new data again.
63  */
64 void KNS_APP_stdln_loop(void);
65 
66 /** @brief Init function needed for the application SW running in front of the GUI
67  *
68  * It initializes the AT command manager and Kineis MAC protocol
69  *
70  * This fct is expected to run once at at each uC's reset (start-up or LPM wakeup), and before the
71  * KNS_APP_gui_loop fct is started/resumed.
72  *
73  * @param[in] context some handler on required HW setting.
74  * @note this handler will be directly sent as is, to MGR_AT_CMD and MCU wrappers. Thus
75  * application can change its type as per needs.
76  */
77 void KNS_APP_gui_init(void *context);

79 /** @brief This fct is the application expected to run in front of the GUI

82  * * Display the firmware version ("AT+FW=?\r\n")
83  * * Display the ID ("AT+ID=?\r\n")
84  * * Transmit some user data ("AT+TX=<USER DATA>\r\n")
85  *
86  * Refer to @ref mgr_at_cmd_page page to get supported AT commands.
87  */
