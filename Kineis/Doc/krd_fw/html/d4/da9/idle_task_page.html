‪IDLE task

IDLE task to be registered in OS

This task is called at last position when there is absolutely nothing else to do in other tasks. Thus, its main purpose is to go into low power mode.

Note
‪When Standby LPM is supported and GUI application is runnin, a 10s delay is added before going to STANDBY in a way to let user to enter a new AT cmd from UART if wanted.
‪In case of Kineis baremetal OS, recheck all queues are empty before LPM, under critical section:
  • ‪STANDBY/SHUTDOWN: can disable all interrupts, uC will re-enable it when entering LPM.
  • ‪SLEEP/STOP: disable all interrupts here, but need to restore it at the very last moment just before entering WFI
