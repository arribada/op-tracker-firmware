8 /**
9  * @page mcu_at_console_page MCU wrapper: AT CMD console
10  *
11  * This console is used to communicate At commands and responses with the device.
12  * Depending on your application need, it could be based on UART link, I2c and so on.
13  *
14  * **Reception**
15  *
16  * On reception side, this wrapper should handle incoming ASCII character as a data stream.
17  * The client of this wrapper should register a callback which will be invoked each time a new
18  * character is received.
19  *
20  * **Transmission**
21  *
22  * On transmission side, it is possible to send strings on same base as classic printf fucntion.
23  * A specific API is also available to send the content of a binary buffer. It shold convert binary
24  * data into ASCII printable strings.
25  */

49 /** @brief Start AT CMD console for AT cmd reception
50  *
51  * It registers client's callback used to treat the received data stream. The callback will be
52  * called each a new character is received.
53  *
54  * @attention This callback may be called under interrupt context. thus, it must not last too long.
55  *
56  * The format of client's callback is:
57  @verbatim
58 bool (*rx_evt_cb)(uint8_t *pu8_RxBuffer, int16_t *pi16_nbRxValidChar));
59  @endverbatim
60  * At each received character, the callback is said to consume data, it updates the buffer pointer
61  * and the number-of-valid-char reduced by what was consummed.
62  * * [in/out] pu8_RxBuffer pointer to the data (update internally if some characters are consummed
63  * * [in/out] pi16_nbRxValidChar number of valid/remaining char
64  *
65  * @param[in] context pointer which may be usefull to configure console (e.g. UART handle)
66  * @param[in] rx_evt_cb pointer to callback invoked at each RX event, i.e. at each received char
67  * @return true if console is correcly started, false otherwise.
68  */
69 bool MCU_AT_CONSOLE_register(void *context,
70  bool (*rx_evt_cb)(uint8_t *pu8_RxBuffer, int16_t *pi16_nbRxValidChar));

72 /** @brief Send AT CMD response to console

74  * This function is used to print AT command's response on AT console.
75  * The prototype is same as printf. Depending on your needs, this function may only implement

78  * @note This could be redirected to printf if available in your application
79  *
80  * @param[in] format refer to printf manual pages
81  */
