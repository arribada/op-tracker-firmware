‪Kineis OS configuration

This page is presenting the configuration of Kineis OS abstraction layer (KNS_OS).

Basically, an OS needs the list of all tasks to be registerd. In kineis OS abstraction layer, this is a simple enumerate containing all handlers (KNS_OS_taskHdlr_t).

For information, tasks are roder from lower priority to upper priority.

Note
‪In can you use the Kineis baremetal OS (USE_BAREMETAL compilation flag), there is actually no notion of priorities on tasks. Priorities are rather handled at queue level. Before popping an event from a queue the baremetal OS checks ther is no event present in higher-priority queues before. Refer to OS queue utilities for extra details on this.
