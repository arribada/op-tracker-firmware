This page is presenting two APPlication task example which exercices the Kineis stack SW (KNS_APP) in a way to send user data to the Kineis network..

Two applications tasks are available:

  • ‪A standalone app (KNS_APP_stdln_loop) is sending one user message (once or 4 retransmit each 60s)
  • ‪A GUI app (KNS_APP_gui_loop) is dedicated to communicate with the Kinéis Device Interface (KDI) through AT commands (AT commands manager) over an UART link (9600 bauds, 8, N, 1).

Those application needs some configurations from user such as:

  • ‪Credentials (ID, ADDR, Secret key)
  • ‪Radio configuration (UL frequency and modulation)
  • ‪Kineis protocol configuration (a.k.a. MAC profile) Refer to "getting started" section for details.

In case of GUI APP, the format of an AT command is like:

  • ‪"<pattern>=?\r\n" (e.g. "AT+PING=?\r\n") to get information
  • ‪"<pattern>=<params>\r\n" (e.g. "AT+TX=<user_data>\r\n") to set information The list of supported AT commands is available through cas_atcmd_list_array.

Communication between tasks (APP and MAC) in done through events posted in OS Queues. By the end, those applications shows how to:

  • ‪Invoke the kineis stack to send USER DATA
  • ‪Wait notification from the Kineis stack once the transmission is completed
