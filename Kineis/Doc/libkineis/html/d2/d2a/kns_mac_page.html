‪MAC task (Kineis Stack entry point)

This page is presenting the task holding the Kineis stack (KNS_MAC) and how to use it

The KNS_MAC_task can be started by the OS as a classical task. Once started, the task will internally go through each queue waiting for any new event coming from the outside world (APP task, KNS_RF events, INFRA interrupts).

The first thing to really start the kineis stack is to init the profile with KNS_MAC_init direct API.

Once initialized, the stack is able to receive envents from the external worlds (APP task, KNS_RF, INFRA ISR).

  • ‪Top APIs APP-to-MAC : cf events in KNS_MAC_appEvtId_t
  • ‪Top APIs MAC-to-APP : cf events in KNS_MAC_srvcEvtId_t
  • ‪Bottom APIs MAC-to-KNS_RF: cf direct APIs kns_rf.h
  • ‪Bottom APIs KNS_RF-to-MAC: cf events in KNS_RF_eventId_t reported to MAC through callbacks
