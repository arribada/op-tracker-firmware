18 /**
19  * @page mgr_log_page LOGGER
20  *
21  * This page is presenting the logging strategy for the kineis stack (\ref MGR_LOG).
22  *
23  * so far, two levels of logging are available: DEBUG, VERBOSE. The logging routines are defined
24  * through static macros.
25  *
26  * Depending on the enabled compilation flags, the source code of the kineis stack will invoke
27  * empty macros (DEBUG=0, VERBOSE=0) or calls to a routine (\ref vMGR_LOG_printf if DEBUG=1 or
28  * VERBOSE=1). This strategy allow to limit the footprint an dimpact on real-time at maximum
29  * when logging is disabled.
30  *
31  * Then, a third static compilation is impacting \ref vMGR_LOG_printf routine:
32  * * if USE_LOCAL_PRINTF is defined, vMGR_LOG_printf is a real function coded in \ref mgr_log.c
33  * * if USE_LOCAL_PRINTF is NOT defined, vMGR_LOG_printf is actually replace by legacy printf
34  * function.
35  *
36  * By defult, in most of the packages delivered by Kineis, this Log Manager is actually outpûting
37  * the log to a dedicated console (via UART).
38  *
39  */
