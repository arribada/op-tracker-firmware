62 /**
63  * @brief get HW settings of the platform concerning RF
64  *
65  * This API is called by Kineis stack to get RF HW settings to reach an expected output power level.
66  *
67  * This is needed to properly set and configure internal/external PA.
68  *
69  * As the settings are specific to HW design, the list of parameters depends of it. E.g. on KRD:
70  * struct rfSettings_t {
71  * bool isRficHPA; // if true use HPA of the STM32WL55 chip, else use LPA
72  * int8_t rficPaLevel; // output power of the RFIC internal PA in dBm
73  * int8_t externalPaGain; // output gain of the external PA in dB
74  * };
75  *
76  * @attention The corresponding driver in Kineis stack will check HW settings are coherent:
77  * * rficPaLevel shall fit in the capabilities of the platform. E.g.:
78  * * KRD_MP: RFIC_PWR_MIN =-17 dBm, RFIC_PWR_MAX = 15 dBm
79  * * KRD_LP: RFIC_PWR_MIN =-9 dBm, RFIC_PWR_MAX = 22 dBm
80  * * The sum of all PA levels shall be equal to the requested input RF level. E.g. in case of KRD:
81  * rficPaLevel + externalPaGain == rf_level_dBm
82  * * KRD_MP: RF_LEVEL_MIN =19 dBm, RF_LEVEL_MAX = 51 dBm (out of spec)
83  * * KRD_LP: RF_LEVEL_MIN =-9 dBm, RF_LEVEL_MAX = 22 dBm
84  *
85  * @param[in] rf_level_dBm Expected output RF level in dBm
86  * @param[in,out] rfSettings Pointer to struct containing settings. Format is void * to be generic
87  * to any platform. Both calle and caller are said to know the real format depending on the platform
88  *
89  * @return Status @ref KNS_status_t
90  */
91 enum KNS_status_t MCU_MISC_getSettingsHwRf(int8_t rf_level_dBm, void* rfSettings);
